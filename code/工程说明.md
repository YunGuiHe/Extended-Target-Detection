# 雷达波形设计工程说明文档 (Code Documentation)

## 一、 项目背景
本项目复现了论文 **《Probabilistically Robust Radar Waveform Design for Extended Target Detection》** 的核心算法与实验。该工程通过 MATLAB 实现了面向扩展目标的“概率鲁棒检测（PRD）”波形设计准则，能够有效抑制因目标冲激响应（TIR）随机闪烁造成的 SINR 剧烈跳动，从而获得稳定的检测性能。

## 二、 文件夹及核心文件结构说明

`code` 目录共包含 1 个系统自测主脚本和 3 个子模块文件夹。所有核心数学模型、推论及迭代寻优代码均模块化解耦，以便于理解和开展二次开发。

---

### 1. 算法核心模块：`algorithms/`
负责波形协方差矩阵的凸差（Dinkelbach）与最小最大化（MM）迭代求解。

| 文件名 | 核心功能 |
| :--- | :--- |
| **`PRD_design.m`** | **主控调度器**。这是波形设计的统一入口（对应论文 Algorithm 3）。它首先调用初始化求解器判断环境（高定序还是低定序），接着选择调用 `solve_high_SINR` 或 `solve_low_SINR` 进行寻找最优协方差矩阵 $\mathbf{R}_s$，最后调用合成模块生成功率受限的波形 $\mathbf{s}^*$。|
| **`baseline_metric17.m`** | **基准对比方法**。实现了文献 [17] 中仅仅追求期望信噪比 $E[\gamma]$ 最大化的经典算法，用于对比评价 PRD 的鲁棒性能优势。|
| **`solve_init_SDP.m`** | **初始化探测器**。由于 Dinkelbach 需可行点起步，该脚本基于线性规划（论文公式 23）初步估测 $\tilde{P}_0$，其核心任务是判断出接下来应该走高 SINR 还是低 SINR 寻优路径。|
| **`solve_high_SINR.m`** | **高 SINR 全局最优求解器**。此时目标函数表现为**拟凸**（论文 Algorithm 1），结合 SDP 提供保证能够收敛到全局最优解（精度高达 $10^{-4}$）的功能。 |
| **`solve_low_SINR.m`** | **低 SINR 次优求解器**。此时目标函数在凸集中无法保证拟凹，故内层循环加入 **MM (Minorization-Maximization) 技术**（论文 Algorithm 2）。利用线性下界近似处理，该代码中包含一个专门求梯度的推导过程，在迭代寻优过程中保证代价不会恶化（次优解）。 |

### 2. 底层工具与模型：`utils/`
封装所有矩阵推导参数及信噪比期望、方差计算函数。

| 文件名 | 核心功能 |
| :--- | :--- |
| **`get_target_params.m`** | $\star$ **【最重要的物理配置】** 定义了论文中的两个扩展目标模型（1 号战斗机 15m / 2 号客机 38m），输出了 TIR 的先验均值向量 $\mathbf{t}_0$ 及协方差矩阵 $\mathbf{R}_t$。|
| **`compute_M.m`** | $M$ 矩阵生成器。构建由波形和移位矩阵等效而成的状态监测矩阵 $M$。这是信噪比二次型的核心。 |
| **`compute_J.m`** | 移位计算模块。通过构造 $\mathbf{J}_q$ 用于将波形沿目标距离分辨率单元切片卷积，构建发射信号的时延拓扑。|
| **`precompute_coeffs.m`** | **预计算加速器**。基于 $F, Z_{num}, Z_B$ 提前算好迭代循环中固定的矩阵分块，可让迭代过程的耗时大幅缩减。|
| **`compute_EV_from_M.m`** | $\star$ **统计特性计算表**。这是数学最核心的一段推导，将信号对卡方分布的无穷积分化简！基于论文 Proposition 1 的解析法，用极简的特征分解，直接求出此刻波形的 $E[\gamma]$（SINR 期望）及 $V[\gamma]$（SINR 方差）。|
| **`synthesize_waveform.m`** | 波形投影工具。因为 SDP 松弛求出的是协方差矩阵 $\mathbf{R}_s$。如果秩不是 1，就得用高斯随机变量（1000次蒙特卡洛抽样）进行惩罚映射，把协方差**还原为实打实的波形序列 $\mathbf{s}^*$**。|
| **`synthesize_waveform_by_Egamma.m`** |（文献[17]基准专属）波形投影工具。功能类似上者，但是选择候选波形的指标换成了仅挑选期望 $E[\gamma]$ 最高的。（PRD 是挑选综合得分 $f$ 最高的）。|

## 三、 用户后续（二次开发）调参指北

方便后期自己进行调节的关键点（所有代码均在顶层结构预留了相关设置）：

### 1. 修改系统硬指标
大部分硬参数定义在每个具体的 `exp_.m` 实验入口的前 20 行：
*   **雷达期望检出信噪比**：`gamma0_dB = 15;` 默认 15dB 是系统给的指标。若想研究极严苛任务，改 20dB（此时由于很难达标，算法2 MM 的用武之地会显著变多）。
*   **恒模要求系数**：`rho = 1/L` 这是最严酷的等边振幅约束 CMC。若放大一些，比如给发射管喘息机会改成 `1/4`，算法搜寻的最优结果分值 $f$ 还会再上一截台阶。（调参见 `exp2`系列）。
*   **蒙特卡洛可靠性**：`N_MC = 1000;`。默认用一千次试探模拟目标。觉得跑图慢且只要验证理论时，可以毫不犹豫地降到 `N_MC = 100`。

### 2. 更改想要探测的目标模型 (Target)
完全解耦在 `utils/get_target_params.m` 内部。
当前：`1`是小截面带有指数衰减模型的战斗机。 `2`是机身为均匀段且三频分散的客机。
*   如果想用自己实验室采集或者电磁场推算的实测反射截面，直接在该函数添加 `case 3`：塞入你这架新飞机的回波占据的格数 `Q` 及其自相关协方差张量矩阵 `Rt` 即可立即适用这个先进的算法框架。

### 3. 主脚本控制台
*   日常验证，建议使用 **`main_run.m`**，将非常耗时的 `exp3` 与 `exp4` 在第 44 - 50 行注释掉，单独调试模块即可。
*   需要所有插图打包进你的研究汇报 PPT / Word 报告时，点击全启运行，该文件夹即会自动吐出全部 5 组实验论文对比图片（`*.png` 格式）。
